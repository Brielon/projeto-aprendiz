Comandos essenciais para a utilização do GIT

Criaçao de um repositorio local com o GIT

A chave SSH deve estar previamente configurada na pasta user/usuario/.ssh

Primeiro devemos nos direcionar para a pasta onde o repositorio sera criado

git init

E um comando unicamente usado para criar um repositorio local no local onde o terminal se encontra(Para se locomover pelas pastas locais usasse "cd" e "dir")

git clone

Se o repositorio ja existir pode se baixar um clone com o comando git clone "e a url do repositorio(SSH)"

git pull

O comando 'pull' carrega todos os arquivos do repositorio global para o repositorio local

Acessando repositorio global via SSH

git config

O comando git config pode ser usado entre outros para configurar o usuario e o email do editor. ex:
"git config user.name"
"git config user.email"

git remote

Gerencia o conjunto de repositorios rastreados:
*-v:mostra os repositorios remotos ja rastreados
*rm "nome":deleta o rastreamento do repositorio indicado
*add nome "URL/SSH":Adiciona um rastreamento com o nome indicado ao repositorio indicado

Manipulando repositorios com o GIT

git status

Iformassoes sobre o estado atual do repositorio

git log

Mostra os comandos feitos

git checkout (branch)

O comando checkout ajuda a se locomover pelas branch's

git checkout -b (branch)

O comando checkout com o parametro -b cria uma copia da branch atual e te transporta para la

git add "."

O comando add adiciona files ao repositorio local "." e uma parametro pra adicionar dados podendo ser substituido pelo nome da file "file.extenção"

git commit

E uma descrição do estado atual da branch, o ideal e que se tenha um commit a cada nova funcionalidade funcional do progeto.

git commit -m "Commit"

Faz o commit apenas das alterações feitas EX: 

{
    "Fechamento da verção 1.1.1-RC1"
    "Lançamento da verção 1.1.1"
    "Iniciado a verção 2.9.0"
}

git revert 'commit'

desfaz o comit indicado

git commit -am

Seria o mesmo que dar um "git add ." em seguida um "git commit -m 'commit_teste'"

git branch

mostra todas as branch's existentes
pode se usar:
git branch "nome_da_Branch": para criar uma nova branch
alem disso pode-se usar:
git checkout -b "nome_da_Branch": p/ criar a branch e ja se posicionar nela

git merge --no-ff origin/1.1.1

Ele mescla a brach atual com a referida, seria para adicionar a funcionalidades ja existentes a verçao nova que esta apenas com as atualizacoes. Ade ter cuidado para nao mesclar a branch atual com uma branch muito antiga pois dessa maeira as funcionalidades das branchs mais atualizadas seriam perdidas

git tag v1.1.1-RC1

Da um nome a branch atual, e aqui que o vercionamento realmente acontece, no caso de rc 1 2 3 etc seriam para projetos que ja passaram pela qualidade porem nao foram aprovados e voltaram para serem corrigidos pela equipe de desenvolvimento.

git push

O comando 'PUSH' faz o upload da tag do repositorio local para o repositorio global

git push --tags

O comando 'PUSH --tags' faz o upload da tag do repositorio local para o repositorio global

git push "nome_do_remote" "nome_da_tag"

para upar uma tag especifica
